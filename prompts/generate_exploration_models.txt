### Regles principals
1. Inspira't en els models bons, però evita còpies trivials.
2. Explora com a màxim 1-2 novetats per model.
3. Prioritza arquitectures simples, entrenables i estables.
4. Explica clarament què canvies, què vols provar i quines serien les següents variants útils.
5. Si dubtes, tria la versió més simple i robusta.

- `description`: resum curt i concret del disseny i la seva intenció.

- `llm_reasoning`: hipòtesi principal, què esperes millorar o comprovar, i per què aquesta proposta és diferent.
- `potential_next_variations`: 2-4 idees concretes per a futures iteracions.

### Guardrails d'arquitectura
- `architecture_definition` obligatori.
- `used_inputs` obligatori i no buit.
- `branches` obligatori i no buit.
- `output_heads` obligatori i no buit.
- `merges` opcional, però si existeix ha de ser llista.
- Cada item de `merges` ha de tenir `name`, `type`, `source_feature_maps` (llista no buida) i `output_feature_map_name`.
- No inventis noms d'inputs fora de `available_inputs_description`.
- No inventis targets fora de `available_outputs_description`.
- `Add`/`Multiply`/`Concatenate` requereixen `input_source_feature_maps` amb >=2 tensors.
- `AttentionKeras`/`MultiHeadAttentionKeras` requereixen tensors de rank >=3.
- `TimeDistributed` només amb inner layer suportada (ex. Dense).
- `Bidirectional` requereix `wrapped_layer` o `inner_layer`.
- `Dense` requereix `units`; `Conv1D` requereix `kernel_size`; `GRU/LSTM` requereixen `units`.

### Claus prohibides (dialecte legacy)
No utilitzis aquestes claus dins `architecture_definition`:
- `inputs`
- `input_layers`
- `shared_backbone`
- `layers` a nivell arrel de `architecture_definition`

### Mini schema obligatori (V3)
`model_definition_full.architecture_definition` ha de tenir aquest patró:
```json
{
  "used_inputs": [
    {
      "input_layer_name": "...",
      "source_feature_name": "...",
      "shape": [800]
    }
  ],
  "branches": [
    {
      "name": "...",
      "input_source_layer": "...",
      "layers": [
        {"type": "Dense", "units": 64, "activation": "relu", "name": "..."}
      ],
      "output_feature_map_name": "..."
    }
  ],
  "merges": [],
  "output_heads": [
    {
      "output_layer_name": "...",
      "maps_to_target_config_name": "...",
      "source_feature_map": "...",
      "units": 1,
      "activation": "linear"
    }
  ]
}
```

Si utilitzes `merges`, cada merge ha de seguir aquest format mínim:
```json
{
  "name": "fusion_main_aux",
  "type": "concatenate",
  "source_feature_maps": ["main_features", "aux_features"],
  "layers_after_merge": [
    {"type": "Dense", "units": 64, "activation": "relu", "name": "fusion_dense"}
  ],
  "output_feature_map_name": "shared_features"
}
```
